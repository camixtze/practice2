# Задание 2.4 — Абстрактный класс vs Интерфейс

## Теоретическое напоминание

| Критерий | Абстрактный класс | Интерфейс |
|----------|-------------------|-----------|
| Отношение | «является» (is-a) | «умеет» (can-do) |
| Наследование | Одиночное (`extends`) | Множественное (`implements`) |
| Поля (состояние) | Могут быть любые | Только `static final` |
| Конструктор | Есть | Нет |
| Методы с реализацией | Да | `default`/`static` (с Java 8) |

**Правило выбора:**
- Если есть общие **поля** (состояние) и отношение «является» → **абстрактный класс**.
- Если нужна **множественная реализация** или отношение «умеет» → **интерфейс**.

## Ваша таблица

| Ситуация | Ваш выбор | Обоснование |
|----------|-----------|-------------|
| Все животные имеют имя и возраст, но звучат по-разному | Абстрактный класс | Есть общее состояние (`имя`, `возраст`) и отношение «является»: кошка является животным, собака является животным |
| Некоторые объекты (документ, изображение, настройки) можно сериализовать | Интерфейс | Это не общий базовый тип по состоянию, а общая способность «умеет сериализоваться». Подходит для разных несвязанных классов |
| Все транспортные средства имеют скорость и топливо, но двигаются по-разному | Абстрактный класс | Есть общее состояние (`скорость`, `топливо`) и отношение «является»: машина является транспортом |
| Робот, посудомоечная машина и телефон — все умеют подключаться к Wi-Fi | Интерфейс | Это общая возможность, а не общий базовый класс. Нужна модель «умеет», плюс такие классы могут быть очень разными |
| Круг, квадрат и треугольник — все фигуры с площадью и периметром, у всех есть цвет | Абстрактный класс | Есть общее состояние (`цвет`) и общее отношение «является»: круг является фигурой, квадрат является фигурой |
| Класс должен одновременно уметь летать, плавать и ездить | Интерфейс | Нужна множественная реализация нескольких независимых способностей: `Flyable`, `Swimmable`, `Drivable` |
