## Вопрос 1

Что выведет этот код? Объясните порядок выполнения блоков.

```java
class Demo {
    static { System.out.print("A "); }
    { System.out.print("B "); }
    Demo() { System.out.print("C "); }
    public static void main(String[] args) {
        System.out.print("D ");
        new Demo();
        System.out.print("E ");
        new Demo();
    }
}
```

**Предсказанный вывод:**

```
A D B C E B C 
```

**Объяснение:**
Объяснение:

Статический блок выполняется при загрузке класса, перед запуском метода main. Поэтому первым выводится "A ".
В методе main выводится "D " — это обычная печать.
При создании первого объекта new Demo(), сначала выполняется конструктор родительского класса (если есть), затем выполняются нестатические блоки и конструктор самого класса. Выводится "B " из нестатического блока и "C " из конструктора.
После этого выводится "E " после первого объекта.
Для второго объекта new Demo(), снова выполняются все те же шаги, выводится "B " и "C " для второго объекта.

## Вопрос 3

Можно ли создать sealed-класс, у которого все наследники — record? Какое преимущество это даёт при использовании в switch?

**Ваш ответ:**
Да, можно создать sealed-класс, все наследники которого будут record. Это преимущество даёт полную гарантированность, что все возможные наследники будут известны на этапе компиляции. Это позволяет использовать паттерн-матчинг с switch без необходимости включать default блок, так как компилятор знает все возможные типы.

## Вопрос 4

Что выведет этот код? Объясните, что такое String Pool и как работает `intern()`.

```java
String a = "Hello";
String b = new String("Hello");
String c = b.intern();
System.out.println(a == b);
System.out.println(a == c);
System.out.println(b == c);
```

**Предсказанный вывод:**

```
false
true
false
```

**Объяснение:**
String Pool — это область памяти, где хранятся строковые литералы для оптимизации. Если строка уже существует в пуле, то новая строка с таким же значением будет ссылаться на тот же объект.
a == b возвращает false, потому что b создаётся через new String(), что создаёт новый объект в куче, а не в пуле.
a == c возвращает true, потому что c вызывается через intern(), что находит строку в пуле и присваивает ссылку на уже существующую строку.
b == c возвращает false, потому что b — это объект в куче, а c — это строка из пула.

## Вопрос 5

Почему этот код не компилируется? Как исправить?

```java
interface A { default void hello() { System.out.println("A"); } }
interface B { default void hello() { System.out.println("B"); } }
class C implements A, B { }
```

**Ваш ответ:**

Код не компилируется из-за конфликта в методах hello() в интерфейсах A и B, так как оба интерфейса предоставляют реализацию этого метода. Для решения проблемы нужно либо:

Переопределить метод hello() в классе C, чтобы решить, какой метод из интерфейсов будет использоваться.
Использовать default метод с разрешением конфликта в классе C.

## Вопрос 6

В чём разница между `StringBuilder` и `StringBuffer`? Приведите конкретный сценарий, когда нужен именно `StringBuffer`.

**Ваш ответ:**

В чём разница между StringBuilder и StringBuffer? Приведите конкретный сценарий, когда нужен именно StringBuffer.

Ваш ответ:
StringBuilder и StringBuffer оба используются для изменения строк, но:

StringBuilder не синхронизирован, то есть не потокобезопасен, но более быстрый.
StringBuffer синхронизирован, то есть потокобезопасен, но медленнее, чем StringBuilder.

StringBuffer нужно использовать в многозадачных (многопоточных) приложениях, когда строки изменяются в нескольких потоках одновременно.

## Вопрос 7

Что выведет этот код? Объясните, что происходит с переменной `x` в лямбде.

```java
int x = 10;
Runnable r = () -> System.out.println(x);
// x = 20;  // что будет, если раскомментировать?
r.run();
```

**Предсказанный вывод:**

```
10
```

**Объяснение:**
Переменная x в лямбде является финальной или эффективно финальной (не изменяется после присваивания). Если раскомментировать строку x = 20;, то код не скомпилируется, потому что лямбда-выражения могут использовать только финальные или эффективно финальные переменные.

## Вопрос 8

Напишите пример лямбда-выражения, которое нельзя заменить на ссылку на метод. Объясните почему.

**Ваш пример и объяснение:**
Пример:

Function<String, String> exclaim = s -> s + "!";

Это выражение нельзя заменить на ссылку на метод, потому что лямбда выполняет операцию конкатенации строки, а не просто вызывает метод. Ссылка на метод может быть использована только в случае прямого вызова метода без дополнительной логики.

## Вопрос 10

Дан `record Point(int x, int y) {}`. Что сгенерирует компилятор автоматически? Можно ли добавить в record изменяемое поле? Можно ли наследоваться от record?

**Ваш ответ:**
EnumMap быстрее HashMap, потому что использует массив для хранения значений, индексируемый по порядковым номерам элементов enum, что даёт доступ за O(1) времени. HashMap использует хеширование и может иметь более сложную реализацию.
